# SLR Parser

В данном проекте реализован SLR-парсер для простейшего арифметического языка.
Операции: `+`, `-`, `*`, `/`, `(`, `)`.

На вход программа получает файл с последовательностью символов, а на выходе выдает визуализацию работы SLR-парсера — стека и буфера. Также в случае ошибки выводится сообщение об этом.

## Разбор языка

Грамматика состоит из 8 продукций:

| № | Продукция |
|---|-----------|
| 1 | `E -> E + T` |
| 2 | `E -> E - T` |
| 3 | `E -> T` |
| 4 | `T -> T * F` |
| 5 | `T -> T / F` |
| 6 | `T -> F` |
| 7 | `F -> id` |
| 8 | `F -> ( E )` |

Для построения расширенной грамматики добавляем: `E' -> E`

Затем строим LR(0)-автомат:

![LR(0)-автомат](assets/LR(0)%20автомат.png)

После этого можем построить таблицы ACTION() и GOTO(), подробно можно посмотреть в:
- [таблицы ACTION и GOTO](assets/Grammar.pdf)

## Сборка и запуск

### Требования
- Компилятор с поддержкой C++17 (gcc, clang)
- CMake (версия 3.10 или выше)
- Flex (лексер)
- Catch2 (для тестов, устанавливается автоматически или через пакетный менеджер)

### Инструкция по сборке

1. Клонируйте репозиторий:
   ```bash
   git clone <url-репозитория>
   cd SLR-parser

2. Создайте директорию для сборки и перейдите в нее:
    `mkdir build`
    `cd build`

3. Запустите CMake для конфигурации:
    `cmake ..`

4. Выполните сборку:
    `make`
    или
    `cmake --build .`

`Compiler` - основная программа,
`unit_tests` - тесты.

### Запуск основной программы

Программа читает входной файл с выражением и выполняет синтаксический анализ.

`./compiler input.txt`

- При успешном разборе выводится пошаговая таблица (если не определён макрос NDEBUG) и сообщение `Parsing succeeded.`

- При ошибке выводится `Parsing failed.`, а также лексическая ошибка, если встречен недопустимый символ.

### Запуск тестов

Для запуска всех тестов выполните:

`./unit_tests`

Тесты проверяют как лексический анализ, так и синтаксический разбор на корректных и некорректных примерах.

### Примечания 

- Для отключения подробного вывода парсера (пошаговой таблицы) можно собрать проект с флагом `-DNDEBUG`.
